Streams

 (require "functional/streams")

This library provides support for lazy lists, aka "streams". It provides the basic functions you need to construct streams and a monad advanced stream comprehensions.

1 Usage

This is how we do it:

1.1 Basics

A stream is a data structure which is either the empty stream, the-empty-stream, or a stream structure, which is a pair with a head, a value of any kind, and a tail, which is a Racket promise, produced with the delay form. Delay results in a promise, which is basically just a thing which will evaluate into a value, later, when you call force. But the point of building streams is so you don’t have to think about this stuff.

You create a stream with stream-cons.

(require functional/streams)
(define one (stream-cons 1 the-empty-stream))

This is a stream with just the value 1 in it. So:

(stream-head one)

Will be one.

(stream-tail one)

Will be the empty stream. Incidentally, the empty stream is itself a promise that produces the empty stream. This behavior may be rethought in the future.

More interesting streams can be built with stream-cons:

(define ones (stream-cons 1 ones))

This produces an infinitely long stream of the value 1.

(stream-head ones)
(stream-head (stream-tail ones))
(stream-head (stream-tail (stream-tail ones)))
...

All evaluate to one. How does this work?

Stream-cons is a special form, rather than a function. It automatically wraps the contents of the second argument in a delay. Eg:

(define ones (stream-cons 1 ones))
(define ones* (stream 1 (delay ones*)))

Are equivalent pieces of code.

2 Functions, Special Forms and Values

(struct stream (head tail-promise))
  head : any/c
  tail-promise : (or/c promise? stream-empty?)

Represents a non-empty stream. This library redefines the stream? predicate created by this structure definition to return true when called on the empty stream as well as stream structures.

the-empty-stream : stream-empty?

Represents a stream with no values in it. When using match, match against the empty stream with (? the-empty-stream).

(stream-empty? s) → (b boolean?)
  s : stream?

Returns true if s is the empty stream, false if s is a stream which isn’t empty, and will produce an error if called with a non-stream.

(stream-tail s) → (result stream?)
  s : stream?

Returns the result of forceing the tail-promise of the stream s, unless s is the empty stream, in which case, this produces an error.

(stream-cons head future)

Constructs a stream, automatically wrapping the expression future in a promise with delay.

ones : stream?

The stream of ones.

zeros : stream?

The stream of zeros.

(stream-from n) → (s stream?)
  n : number?

This function returns a stream which starts at n and increases by one perpetually.

(stream-down-from n) → (s stream?)
  n : number?

This function returns a stream which starts at n and decreases by one perpetually.

(stream-range start incr terminal) → (s stream?)
  start : number?
  incr : number?
  terminal : number?

Produces a stream over a range of numbers. The stream starts with start, increments by incr, and never exceeds or reaches the value terminal.

If incr is negative, the stream goes down and never falls below or equals the value terminal.

(take s n) → (l list?)
  s : stream?
  n : number?

Convert a stream into a list of n elements (or fewer, if the stream is shorter) by taking the first n elements and placing them in a list.

(stream-map f s ...) → (s stream?)
  f : procedure?
  s : stream?

Takes a function of N arguments and N streams and produces a new stream which contains (f stream-el1 ... stream-eln).

(stream-zip s ...) → (s stream?)
  s : stream?

Return a stream of lists of elements of the input streams. As long as the shortest input stream.

(stream-append s1 s2) → (s stream?)
  s1 : stream?
  s2 : stream?

Takes two streams and returns their concatenation. The identity operator when s1 is an infinite stream.

(stream-interleave s1 s2) → (s stream?)
  s1 : stream?
  s2 : stream?

Takes two streams and returns a stream which is their elements, interleaved.

(stream-add-tail s1 tail) → (s stream?)
  s1 : stream?
  tail : promise?

Produces a stream which has the elements of s1 but whose tail is the stream produced by tail, a promise to produce a stream.

(stream-map-cat f s) → (s stream?)
  f : procedure?
  s : stream?

Takes a function f, which transforms an element of s into a stream, and returns the concatenation of all these streams.

(stream-map-interleave f s) → (s stream?)
  f : procedure?
  s : stream?

Takes a function f, which transforms an element of s into a stream, and returns the interleaving of all these streams.

(stream-bind s f) → (s stream?)
  s : stream?
  f : procedure?

Same as stream-map-cat with argument order reversed. Bind for the stream monad.

(stream-bind* s f) → (s stream?)
  s : stream?
  f : procedure?

Same as stream-map-interleave with argument order reversed. Bind for the interleaving stream monad.

(stream-return item) → (s stream?)
  item : any/c

Returns a stream with one element, item.

the-stream-monad : monad?

The stream monad.

the-stream-monad* : monad?

The interleaving stream monad.

(list->stream lst) → (s stream?)
  lst : list?

Returns a stream with the same elements as lst. Lazy, so that side effects on lst will show up in the stream when it is evaluated, except for the first element.

(list->cycle lst) → (s stream?)
  lst : list?

Returns a stream which has the elements of lst repeating forever.

(stream-of value) → (s stream?)
  value : any/c

Return an infinite stream of value.

(stream-of-random-ints z w) → (s stream?)
  z : integer?
  w : integer?

Return a stream of random integers (32 bit) from a simple random number generator seeded with z and w. Each stream of numbers is repeatable.

(stream-of-random-floats z w) → (s stream?)
  z : integer?
  w : integer?

Return a stream of random floats on the open interval (0,1).

(stream-of-normals z w) → (s stream?)
  z : integer?
  w : integer?

Returns a stream of normally distributed numbers with mean zero and standard deviation one.
